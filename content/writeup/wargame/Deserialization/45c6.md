2. 뽑은 값과 결과값 동일해야 승리: 아래와 같이 결과값 결정

```php
$results = draw(45, 6);
```

5. 목표: 내가 뽑은 값과 result.php가 뽑을 값을 같게 만들기
6. 그러나 서버 draw는 랜덤 -> 값을 예측해서 맞추는 건 불가능

8. PHP 직렬화는 참조(reference)도 표현함 -> `R:n`
   - numbers를 results의 참조로 만들면 둘이 같은 배열 하나를 가리킴
9. 서버가 `$ticket->results[$i] = $results[$i]`로 draw를 쓰는 순간 numbers도 같이 바뀜
   - 비교되는 두 대상이 물리적으로 동일 -> 서버가 뭘 뽑든 항상 일치
10. 페이로드 생성 (로컬 php 없으면 docker `php:8.1-cli`)

    ```php
    $t = new Ticket();
    $t->results = [1,2,3,4,5,6];
    $t->numbers = &$t->results;   // numbers를 results의 참조로
    ```

11. 이 base64를 `ticket` 쿠키 값으로 넣고 result.php 요청
    - 서버가 unserialize하며 참조 관계까지 복원 -> numbers와 results가 서버 안에서 같은 배열 공유

## 배운 점
- 직렬화/serialize(): 메모리 속 객체를 문자열로 누른 다음, 나중에 다시 객체로 되살릴 수 있게 만들기
- base64: 문자열을 안전한 문자로 포장
	- 보낼 때:  객체 → serialize() → 문자열 → base64_encode → 쿠키
	- 받을 때:  쿠키 → base64_decode → 문자열 → unserialize() → 객체
